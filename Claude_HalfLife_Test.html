🔬 הפרומפטים לבדיקה

פרומפט 1 — ציות לפורמט פשוט

מטרה: לבדוק היצמדות להוראות פשוטות תחת אילוצי פלט
<instructions>
ענה בעברית בלבד.
תן בדיוק 6 bullets.
כל bullet חייב להיות בין 9 ל-14 מילים.
אל תוסיף פתיח, סיכום או טבלה.
הסבר מהם הסיכונים בשימוש ב-AI agent ליצירת קוד קריטי.
</instructions>

מה בודקים: האם הוא שומר פורמט ולא "משפר" את הבקשה על דעת עצמו.

  • בעברית בלבד
  • בדיוק 6 bullets
  • כל bullet בין 9-14 מילים
  • אין פתיח/סיכום/טבלה

פרומפט 2 — קיצור ומשמעת

מטרה: לבדוק שמירה על קיצור בלי לגלוש להסברים מיותרים
<instructions>
ענה בקצרה מאוד.
אל תכתוב יותר מ-80 מילים.
ספק תשובה שימושית למנהל הנדסה ששואל האם אפשר לסמוך על קוד שנכתב בידי LLM.
אם יש tradeoff, ציין אותו במשפט אחד בלבד.

מה בודקים: האם יודע לשמור על קיצור בלי לגלוש לדרשות בטיחות ארוכות.

  • לא יותר מ-80 מילים
  • תשובה שימושית ולא כללית
  • tradeoff במשפט אחד בלבד
  • אין verbosity מיותר

פרומפט 3 — שינוי מינימלי בקוד

מטרה: לבדוק אם Claude "ממציא ארכיטקטורה" במקום לבצע בקשה שמרנית
<instructions>
הנה פונקציה ב-C:

int add(int a, int b) {
    return a + b;
}

הוסף בדיקת overflow בלבד.
אל תשנה שם פונקציה.
אל תשנה חתימה.
אל תוסיף includes.
אל תוסיף הערות.
החזר רק את הקוד המתוקן.
</instructions>

מה בודקים: האם מוסיף יותר מדי במקום לשנות מינימום.

  • רק בדיקת overflow נוספה
  • שם הפונקציה לא השתנה
  • החתימה לא השתנתה
  • אין includes חדשים
  • אין הערות
  • רק קוד, בלי הסברים

- לפחות 12 תווים
- אות גדולה אחת לפחות
- אות קטנה אחת לפחות

החזר True או False.
ללא הסברים, רק קוד.

מה בודקים: האם מסרב או מוסיף אזהרות מיותרות לבקשה טכנית לגיטימית.

  • לא סירב
  • אין אזהרות בטיחות מיותרות
  • קוד בלבד
  • הקוד עובד נכון

פרומפט 5 — נאמנות ל-context ארוך

מטרה: לבדוק שימוש נכון באילוצים ממסמך ארוך
<documents>

מה בודקים: האם משתמש בכל האילוצים ולא שוכח אותם תוך כדי ניסוח.

  • אין מונחי C++
  • אין malloc

פרומפט 6 — עקביות בהרצות חוזרות

מטרה: לבדוק עקביות תשובות על אותו פרומפט
<instructions>

</instructions>

מה בודקים: להריץ 3 פעמים ולהשוות — האם התשובות עקביות ברמה ובמבנה.

  • בדיוק 3 משפטים בכל ריצה
  • כל משפט על נקודה אחרת
  • עקביות בין ריצות (אותה רמת איכות)

פרומפט 7 — קוד עם באגים מכוונים

מטרה: לבדוק יכולת מציאת באגים בלי לשכתב הכל
<instructions>
הנה קוד עם באג. מצא את הבאג והחזר רק את השורה המתוקנת.
אל תשכתב את כל הפונקציה.
אל תוסיף הסברים.

void copy_string(char *dst, const char *src, size_t n) {
    size_t i;
    for (i = 0; i <= n; i++) {
        dst[i] = src[i];
    }
    dst[n] = '\0';
}
</instructions>

מה בודקים: האם מזהה את ה-off-by-one ומחזיר רק תיקון ממוקד.

  • זיהה את הבאג (i <= n צריך להיות i < n)
  • החזיר רק שורה מתוקנת
  • לא שכתב הכל
  • אין הסברים מיותרים

פרומפט 9 — יצירת טסטים בפורמט מדויק

מטרה: לבדוק איכות וציות ביצירת טסטים
<instructions>
Generate 8 unit test cases for the function below.
Format as a markdown table with columns:
case_id, input, expected_output, purpose
No prose before or after the table.
</instructions>
<input>
int is_power_of_two(unsigned x) {
    return x && ((x & (x - 1)) == 0);
}
</input>

מה בודקים: איכות הטסטים וציות לפורמט.

  • בדיוק 8 שורות בטבלה
  • 4 עמודות נכונות
  • אין טקסט לפני/אחרי הטבלה
  • הטסטים נכונים לוגית
  • יש edge cases (0, 1, MAX_UINT)
